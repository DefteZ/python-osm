python-osm documentation

python-osm is a set of classes an tools for openstreetmap data and files.

src/osm/pyosm

This is the base class that contains the data model of python-osm. It also contains a xml parser to read osm files.

When called on commandline with an osm file as argument, the class prints a statistic of the file. 
werner@linux-g0e5:~/osm/src/python-osm/src/osm> ./pyosm.py Rottweil.osm 
filename: Rottweil.osm
  Nodes:     3020
  Ways:      6
  Relations: 1

src/osm/multipolygon.py

This class can be used to work with multipolygon relations. Beside the commandline options you can perform node in multipolygon checks.

multipolygon commandline

When called on command line interface you can either check multipolygons for way gaps or create polygon files for osmosis. 
usage: multipolygon.py --relation=ID [options]
load a multipolygon from the OSM-API or from an OSM file
export osmosis boundary polygon or check the multipolygon for errors
  -h, --help: print this usage message
  -i, --infile: osmfile to load
  -r, --relation: multipolygon relation id
  -m, --osmosispolygon: outfile for osmosis boundary polygon

multipolygon commandline examples

Load a multipolygon with the OSM api and check it.

tools/relation2gpx.py

A tool to download relations from the OSM API and create gpx files. Each relation becomes a track (trk) and each way a track segment trkseg. The segments are not ordered.

relation2gpx.py commandline

usage: relation2gpx.py [-o GPXFILE] -r relation1,relation2,...
load relations from the OSM API and create gpx files
  -h, --help: print this help message
  -o, --outfile: specify the filename of the gpx file
  -r, --relations: comma seperated list of relations

relation2gpx.py examples

Get relation 157289 (HW4 hiking route) and write it to default filename out.gpx 
werner@linux-g0e5:~/osm/src/python-osm/tools> ./relation2gpx.py -r 157289
